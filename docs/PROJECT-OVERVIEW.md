# DELPHI (NL-COP) 프로젝트 개요

---

## 목적

**"오늘 바로 쓸 수 있는 한국형 지휘통제 보조 AI"**

DELPHI는 한국군 망분리 환경에 최적화된 **다중출처 정보융합 + 베이지안 추론 지휘통제 대시보드**입니다.

한국군의 십여 개 분리된 작전/정보망에서 산발적으로 들어오는 첩보(위성영상, 신호정보, 인간정보, 공개출처 등)를 **자동으로 융합**하고, 룰베이스 베이지안 엔진이 **어떤 첩보가 발사 확률을 몇 % 끌어올렸는지** 정량적 근거를 완벽히 설명하여, 지휘관이 블랙박스가 아닌 **통계로 증명된 판단**에 기반해 타격 결심을 내릴 수 있게 합니다.

---

## 해결하고자 하는 문제

### 현실: "정보가 너무 늦다"

한국군은 보안을 위해 십여 개의 작전/정보망으로 분리되어 있으며, 모든 정/첩보 융합은 철저히 **사람의 수작업**에 의해 이뤄지고 있습니다. 지휘관들의 가장 큰 고충은 이 수작업으로 인한 판단 지연입니다.

### 왜 팔란티어로 해결할 수 없는가?

팔란티어는 **통합 클라우드 환경의 미군**을 전제로 설계되었습니다. 망분리된 국군 환경에는 맞지 않습니다. 그래서 저희는 **추론 엔진 자체를 독자 개발**했습니다.

### 왜 단순 LLM으로 해결할 수 없는가?

> "설명할 수 있어야 믿을 수 있다"

GPT나 Claude 같은 LLM은 "북한이 미사일을 쏠 것 같다"라고 추측은 하지만, **왜 그런 결론이 나왔는지 수학적으로 설명하지 못합니다.** 환각(Hallucination) 리스크 때문에 생사가 오가는 군사 작전에 의존할 수 없습니다.

DELPHI의 베이지안 엔진은 **없는 것을 지어내지 않습니다.** 철저한 군사 매뉴얼(규칙)을 따르며, 어떤 첩보 때문에 확률이 몇 % 올랐는지를 명확한 숫자로 증명합니다.

### 문제 → 해결 매핑

| 기존 문제 | DELPHI 해결 방식 |
|-----------|-----------------|
| 십여 개 망 분리 → 수작업 융합 → 판단 지연 | 단일 observation 테이블로 8개 출처 자동 통합 |
| LLM 환각 → 군사 작전에 신뢰 불가 | 룰베이스 베이지안 엔진 (환각 원천 차단) |
| "왜 99%인지" 설명 불가 (블랙박스) | 첩보별 확률 기여도 정량 제시 (감사 가능) |
| 불확실성 미반영 (확신 편향) | SPUQ 기반 신뢰도 가중치 적용 |
| 과거 사례 참조 어려움 | 303건 역사적 발사사례 RAG 검색 |

---

## 핵심 철학: "설명할 수 있어야 믿을 수 있다"

지휘관이 분석관을 신뢰하는 이유는, 분석관이 자신의 판단 근거를 명확히 설명할 수 있기 때문입니다.

DELPHI는 이 원칙을 AI로 구현합니다:

- 텍스트 보고서 → 온톨로지(지식그래프)로 매핑 → 엔진이 전장 상황을 엄밀하게 이해
- 시간에 따라 산발적으로 들어오는 첩보들이 **어떻게 발사 확률을 99%까지 끌어올렸는지** 인과관계를 정량적으로 시각화
- 지휘관은 단순히 99%라는 숫자만 보는 것이 아니라, **어떤 첩보들이 모여 이 99%를 만들었는지 그 수학적 기여도(가중치)를 한눈에 확인**

## 기술 스택

### Frontend

- **Next.js 14** (App Router) + **React 18** + **TypeScript**
- **CesiumJS** — 2D/3D 지도 시각화 (마커, 카메라 추적)
- **Tailwind CSS** — UI 스타일링
- **React-Markdown** — AI 응답 렌더링

### Backend

- **FastAPI** (Python 3.11+) + **Uvicorn**
- **Supabase** (PostgreSQL + PostGIS + pgvector)
- **Redis/Upstash** — 추론 결과 캐싱

### AI/ML

- **베이지안 추론 엔진** (`lib/bayesian.ts`) — 가설별 사후확률 계산
- **SPUQ** — 소형 LLM 다중 샘플링 기반 불확실성 정량화
- **RAG** — pgvector 임베딩 유사도 검색 (발사사례 303건)
- **OpenAI / Anthropic Claude** — 코파일럿 채팅 및 브리핑 생성

### 데이터베이스 스키마 (주요 테이블)

| 테이블 | 용도 |
|--------|------|
| `observation` | Layer 1 원시 관측 (통합 단일 테이블) |
| `facility_ontology` | 북한 시설 온톨로지 |
| `missile_ontology` | 무기체계 제원 |
| `emitter_ontology` | 신호정보 방사체 분류 |
| `orbat` / `blue_orbat` | 적/아군 편성 |
| `launch_cases` | 역사적 발사사례 (RAG용) |

## 아키텍처 (4계층 파이프라인)

```
[입력] 분석관 원시보고 (IMINT, SIGINT, HUMINT, OSINT, UAV, MASINT, GEOINT, CYBINT)
         │
         ▼
[구조화] 소형 LLM + SPUQ → 행동분류(8종) + 구조화 필드 추출 + 신뢰도 산출
         │
         ▼
[추론] 베이지안 엔진 → P(H|E₁,E₂,...) ∝ P(H) × ∏ P(Eᵢ|H) × w(cᵢ)
         │
         ▼
[보고] 대형 LLM + RAG → 자연어 브리핑 + 증거 추적성 + 교차검증
```

## 주요 기능

1. **실시간 지도 시각화** — MGRS→WGS84 좌표 변환, 출처별 색상 마커, 시나리오 위치 추적
2. **7단계 시나리오 타임라인** — 개발→연료→이동→NOTAM→임박→발사→검증, 10단계 H-0 스크러버
3. **적 위협 패널** — 가설별 확률 막대, 연료유형/사거리/발사시간 신뢰도, 기여증거 표시
4. **아군 태세 패널** — ROK/US 부대 위치, 방어진지, 추적 상태
5. **AI 코파일럿 채팅** — 온톨로지 기반 엔티티 해석 + RAG 사례검색 + 컨텍스트 그라운딩
6. **증거체인 추적** — 관측 ID → 가설 기여도 역추적 (설명가능한 AI)

## 차별점

- **패턴 매칭이 아닌 확률적 추론** — 진정한 베이지안 네트워크 기반 위협 평가
- **설명가능성 (Auditability)** — 모든 확률 판단에 대해 "어떤 첩보가 몇 % 기여했는지" 역추적 가능
- **환각 원천 차단** — LLM에 판단을 맡기지 않고, 룰베이스 엔진이 군사 매뉴얼에 따라 추론
- **한국군 맞춤** — 팔란티어와 달리 망분리 환경에서 동작 가능한 독립형 아키텍처
- **SPUQ 불확실성 정량화** — 저신뢰 증거의 과대 반영 방지
- **실시간 스트리밍** — 관측 도착 즉시 사후확률 갱신 및 시각화
