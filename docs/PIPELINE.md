# NL-COP 베이지안 융합 추론 파이프라인

## 전체 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         입력 계층 (Input Layer)                               │
│  판독관 행위 보고서 (비정형/정형, 러프한 확신도 포함)                              │
└──────────────────────────────────┬──────────────────────────────────────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                    정형화 계층 (Structuring Layer)                             │
│  소형 LLM + SPUQ 불확실성 추정                                                │
│  - 액션 클래스 분류                                                           │
│  - 정형 필드 기입                                                             │
│  - 베이즈 추론용 확신도 산출                                                    │
└──────────────────────────────────┬──────────────────────────────────────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                    추론 계층 (Inference Layer)                                 │
│  베이지안 추론 엔진                                                            │
│  - 입력: 사전 지식(가설 DB) + 정형화된 징후(액션 클래스)                           │
│  - 출력: 각 가설 조합의 사후확률 + 불확실성                                       │
└──────────────────────────────────┬──────────────────────────────────────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                    보고 계층 (Reporting Layer)                                 │
│  대형 LLM + RAG                                                              │
│  - 추론 엔진 출력 + 사전지식 + 개별 액션 클래스 (저수준 입력)                       │
│  - 종합 보고서 생성, 근거 추적, 출처 표기                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 1. 입력 계층: 판독관 행위 보고서

판독관이 작성한 Action 단위의 보고서. 비정형 텍스트 또는 반정형 양식.

**포함 정보:**
- 관측 내용 (자연어)
- 관측 시각
- 출처 유형
- 판독관의 러프한 확신도 (예: "확실", "추정", "불확실")

## 2. 정형화 계층: 소형 LLM + SPUQ

### 역할
Raw 보고서를 미리 정의된 **액션 클래스**로 분류하고, 정형화된 필드에 정보를 기입.

### SPUQ (Sampling-based Predictive Uncertainty Quantification)
- 소형 로컬 LLM의 환각과 판단 불확실성을 줄이기 위한 기법
- 동일 입력에 대해 N회 샘플링 → 응답 분포로 불확실성 추정
- 클래스 분류 불확실성과 필드 기입 불확실성을 별도 산출
- 이 불확실성을 베이지안 파이프라인의 likelihood에 반영

### 중간표현 액션 클래스 목록

| 클래스 | 설명 | 주요 필드 |
|--------|------|-----------|
| **영상자산 (IMINT)** | 위성/항공 영상 판독 결과 | 좌표, 객체유형, 변화탐지, 시각 |
| **첩보자산 (HUMINT)** | 인적 정보원 보고 | 정보원 신뢰도, 내용, 확인여부 |
| **신호자산 (SIGINT)** | 통신/전자 신호 수집 | 주파수, 패턴, 발신원, 내용요약 |
| **지리정보자산 (GEOINT)** | 지형/시설 공간분석 | 좌표, 시설유형, 변화내역 |
| **공개출처 (OSINT)** | 뉴스/SNS/기념일/훈련일정 | 출처URL, 날짜, 키워드, 맥락 |
| **계측자산 (MASINT)** | 적외선/음향/진동/화학 탐지 | 센서유형, 측정값, 위치, 시각 |
| **사이버자산 (CYBINT)** | 네트워크/사이버 활동 징후 | 대상시스템, 활동유형, 지표 |
| **기상/환경 (WXINT)** | 발사/작전 관련 기상조건 | 풍속, 시정, 기온, 적합도 |

### 각 액션 클래스 공통 필드

```typescript
interface ActionClass {
  id: string;
  classType: ActionClassType;
  timestamp: string;          // 관측 시각
  source: string;             // 출처 식별자
  rawReport: string;          // 원본 보고서 텍스트
  confidence: number;         // 0-1, SPUQ 기반 분류 확신도
  fieldUncertainty: Record<string, number>; // 각 필드별 불확실성
  analystConfidence: number;  // 판독관 원래 확신도
  fields: Record<string, any>; // 클래스별 정형 필드
}
```

## 3. 추론 계층: 베이지안 추론 엔진

### 사전 지식 (Prior Knowledge)

과거 사례를 기반으로 구축된 **"징후 → 결과" 인과 매핑**.

```
특정 액션 조합 → 특정 가설 (결과)
```

**예시:**
- 액체 산화제 이동 + 고각 발사대 이동 → 액체 장거리 발사체 발사
- TEL 이동 + 단시간 준비 + 연료주입 징후 없음 → 고체 단거리 발사체

### 가설 구조

가설은 계층적 트리 형태로 구성:

```
미사일 발사 가설
├── 연료 유형
│   ├── 고체 발사체
│   └── 액체 발사체
├── 사거리
│   ├── 단거리 (< 1000km)
│   ├── 중거리 (1000-5500km)
│   └── 장거리 (> 5500km)
└── 발사 시점
    ├── 임박 (< 2시간)
    ├── 준비중 (2-24시간)
    └── 초기 징후 (> 24시간)
```

### 베이지안 추론 수식

```
P(H|E₁,E₂,...,Eₙ) ∝ P(H) × ∏ P(Eᵢ|H) × w(cᵢ)
```

- `P(H)`: 가설의 사전확률 (과거 사례 빈도 기반)
- `P(Eᵢ|H)`: 가설 H가 참일 때 징후 Eᵢ가 관측될 우도 (사전지식 DB)
- `w(cᵢ)`: SPUQ 불확실성 가중치 (확신도 낮으면 약한 증거로 처리)

### 핵심 원칙

> **사전 가설은 과거 기록을 바탕으로 미리 구축되는 판단의 베이스라인.**
> "징후 → 결과" 인과 관계가 성립된다면 미사일 뿐만이 아닌 어떠한 것도 포착 가능.
> 미래 예측이 아닌 **현재 상태의 추론**도 가능.

### 출력

```typescript
interface InferenceResult {
  hypotheses: {
    id: string;
    name: string;
    posterior: number;      // 사후확률
    uncertainty: number;    // 불확실성
    evidenceChain: string[]; // 근거 액션 ID 목록
  }[];
  topHypothesis: string;
  overallConfidence: number;
  updatedAt: string;
}
```

## 4. 보고 계층: 대형 LLM + RAG

### 역할
추론 엔진의 정량적 출력을 인간이 이해할 수 있는 보고서로 변환.

### 입력 (RAG 형태)
- 추론 엔진 출력 (가설 확률, 불확실성)
- 사전지식 DB (관련 가설/사례)
- 개별 액션 클래스 인스턴스 (저수준 원본 데이터)

### 기능
1. **종합 보고**: 추론되는 징후 및 상태를 종합해서 보고
2. **근거 추적**: 어떤 징후의 근거를 물어보면:
   - 추론 엔진의 추론 과정 설명
   - 입력 데이터의 액션 클래스 조회
   - 원본 보고서까지 추적
   - 출처 표기 및 추론 과정 설명

## 5. 실시간 업데이트

> 이 전체 파이프라인은 **입력 데이터가 추가될 때마다 업데이트**.

- 새 보고서 입력 → 정형화 → 베이지안 사후확률 갱신 → 보고서 재생성
- 점진적 업데이트 (전체 재계산 불필요, 새 증거만 반영)
- 확률 변화가 임계값 초과 시 알림 트리거

## 시스템 데이터 흐름 요약

```
판독관 보고서
    │
    ▼ [소형 LLM + SPUQ]
액션 클래스 인스턴스 (확신도 포함)
    │
    ├──────────────────────────┐
    ▼                          ▼
베이지안 추론 엔진         사전 지식 DB
(사후확률 계산)            (과거 사례 기반)
    │
    ▼
가설 확률 분포
    │
    ▼ [대형 LLM + RAG]
종합 보고서 (근거 추적 가능)
```
